Binary representation
  • Binary system uses two digits: 0 and 1.
  • Each digit (bit) represents a power of 2.
  • Example:
  • Binary 1011 = 1·23 +0·22 +1·21 +1·20 = 11.
Why Binary?
  • Simple to implement in digital circuits
  • High reliability due to clear distinction between states (0 and 1).
  • Compatibility with Boolean logic for computation and storage.
Using Integer Representation
  • Represents only positive integers.
  • Range: 0 to 2n −1 for n-bit numbers.
  • Example
  • 8-bit number: Range is 0 to 28 −1 =255.
Signed Integer Representation
  • Most significant bit (MSB) is the sign bit:
  • 0 for positive, 1 for negative.
  • Representation methods
    1. Sign-and-magnitude
    2. 1’s complement
    3. 2’s complement (most common).
1’s Complement Representation
  • In 1’s complement representation, the representation of the positive number is same as the negative number. But the representation of the negative number is different.
  • To represent a negative number, take corresponding positive number and invert all bits.
2’s Complement Representation
  • Widely used for signed integers. Simplifies arithmetic operations.
  • How to compute a negative number:
    1. Take the binary of the positive number.
    2. Invert the digits (1’s complement).
    3. Add 1 to the result
  • Example: −5 in 4-bit:
    1. Positive 5: 0101.
    2. 1’s complement: 1010.
    3. Add 1: 1011.
